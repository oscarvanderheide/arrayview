ARCHITECTURE SUMMARY
==========================================================

Every enter/exit function has a sync block that populates modeManager:
  - modeManager.currentViews[] — thin View wrappers over legacy canvases
  - modeManager.modeName       — 'normal'|'multiview'|'compare'|...'mip'
  - modeManager.currentLayout  — the active LayoutStrategy instance

modeManager public API:
  getViewUnderMouse()  → View | null   (for per-pane hover targeting)
  getFocusedView()     → View | null   (for keyboard command dispatch)
  getViewById(id)      → View | null   (for diff/registration center views)
  currentViews[]       → all active views with displayState and colorBar
  modeName             → string (current mode)
  LAYOUT_REGISTRY      → { 'normal': () => new NormalLayout(), ... }
