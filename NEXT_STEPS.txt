PHASE 12 — Keybind Collapse
Status: BLOCKED — REQUIRES HUMAN DECISION

  The plan says to replace command bodies with view.method() calls:
    commands['logscale.toggle'].run = () => {
        const view = modeManager.getFocusedView();
        view?.toggleLogScale();
    }

  The problem: View.render() still delegates to _legacyUpdateView(), which
  reads legacy globals (logScale, colormap_idx, complexMode, etc.) — NOT
  displayState. So calling view.toggleLogScale() updates displayState.logScale
  but has no effect on rendering. Nothing would visually change.

  To unblock Phase 12:
    Option A: Wire the render pipeline to read from displayState. This is the
              correct long-term fix but is a large change not spec'd in Phases 1-17.
    Option B: Make view.toggleLogScale() also dual-write to the legacy global.
              Quick fix: adds view.toggleLogScale() to the Phase 2 pattern.
    Option C: Skip Phase 12 entirely and do it in a future refactor cycle.

  Recommendation: Option B (dual-write). For each View method, also write the
  legacy global so _legacyUpdateView() picks it up. This extends the shim
  pattern from Phase 2 to all action methods.

PHASE 17 — Final Cleanup
Status: BLOCKED — requires Phase 12 first

  Phase 17 deletes the shim layer and legacy globals. Can't do that until the
  render pipeline reads from displayState (Phase 12 prerequisite above).

==========================================================
ARCHITECTURE SUMMARY
==========================================================

Every enter/exit function now has a sync block at the end that populates
modeManager.currentViews and sets modeManager.modeName. The modeManager
provides:
  - getViewUnderMouse()    — for per-pane hover targeting
  - getFocusedView()       — for keyboard command dispatch
  - getViewById(id)        — for diff/registration center views
  - currentViews[]         — all active views with displayState and colorBar
  - modeName               — 'normal'|'multiview'|'compare'|'qmri'|'mip'|...

The render pipeline (View.render() → _legacyUpdateView()) still reads from
legacy globals. The shim layer (Section 7) keeps manualVmin/manualVmax
synchronized between the two systems.
